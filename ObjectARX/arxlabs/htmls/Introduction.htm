ObjectARX 2000 Training

Introduction

Welcome to ObjectARX 2000 Training. The tutorial consists of a number of predefined steps that must be completed in order. Step 1 outlines the requirements for setting up an ObjectARX project manually. Steps 2 through to Step 7 make use of the ObjectARX Wizard and each step is based upon the previous step. When you have completed the tasks outlined in a particular step, that step then becomes the starting point for the next step.

Before you begin you must have installed the following:

  • AutoCAD 2000
  • ObjectARX 2000
    • http://www.autodesk.com/objectarx
  • Microsoft Visual C++ 6.0 (Service Pack 2 or later)
  • ObjectARX 2000 Wizard.
  • Windows NT 4.0 (Service Pack 2 or later), Windows 2000.

    • NOTE: Windows 95/98 is not recommended as a development environment for ObjectARX.

    The tutorial assumes that you have a knowledge of AutoCAD and Visual C++. You should complete each step yourself, but, if you get into difficulty, you can use the ObjectARX Training project file.  The code contain in folder 'Step03' for example represents the starting point for the code developed in Step 3 of the tutorial. If you want to see the finished code for the tasks outlined in Step 3 of the tutorial, look at the code contained in the folder 'Step04'. This scheme is repeated for all the other steps of the tutorial. Table of Contents

  • Step 2 - Input Functions
      • User input, custom commands and creating ObjectARX applications using the ObjectARX Wizard.
  • Step 3 - Symbol Tables

  • Step 4 - Xrecords - Named Objects Dictionary
      • Creating Xrecords and manipulation of the Named Objects Dictionary.
  • Step 5 - Custom Objects - Extension Dictionary
      • Create a custom class derived from AcDbObject, store the custom class in a block reference's Extension Dictionary.
  • Step 6 - Custom Entities
      • Create a custom entity derived from AcDbEllipse, store user input data an integral part of the custom entity.
  • Step 7 - Transient Reactors
      • Explore the use of editor, object and database reactors used to control events on a block reference entity.
